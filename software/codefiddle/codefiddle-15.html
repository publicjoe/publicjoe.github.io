In this part of the tutorial, we will add auto-closing tags to the HTML editor.

Auto-closing Tags

Ideally when the user types an opening tag, the closing tag should be automatically inserted. This happens in most code editors, but not in our CodeFiddle yet. So how are we going to approach this? Monaco supports auto-closing tags, but it's not enabled by default. So we need to enable it. First of all lets add the code to enable auto-closing tags to our app.js file using the built in features of Monaco. We will also add a little bit of formatting to the HTML editor. Since we are using the CDN loader, we will set these options inside the require(['vs/editor/editor.main'], ...) callback, ideally right before we create the editor instance.

app.js 
require(['vs/editor/editor.main'], () => {

  // --- Configure HTML Auto-Closing ---
  monaco.languages.html.htmlDefaults.setOptions({
    format: {
      tabSize: 2,
      insertSpaces: true
    },
    suggest: {
      html5: true
    },
    // This enables the "Auto Close" behavior
    autoClosingTags: true 
  });

The we can add the following code to the app.js file to enable auto-closing tags.

app.js 
editors.html = monaco.editor.create(document.getElementById('html-editor'), {
  value: "
Welcome to CodeFiddle!
",
  language: 'html',
  theme: 'vs-dark',
  automaticLayout: true,
  // Ensure the individual editor instance also honors the setting
  autoClosingTags: 'always' 
});

Go ahead and try it!

Failure!

So what should have happened. Well, when the user types an opening tag, the closing tag should be automatically inserted. For instance, when the user finishes typing <div>, the editor automatically adds </div>. This is not the case with our CodeFiddle.

After doing some research, I found that the auto-closing tags are not enabled by default in Monaco.

While VS Code (which uses Monaco) has this feature, it is actually powered by a VS Code-specific extension that isn't bundled into the core monaco-editor library we are using. The property autoClosingTags: 'always' is technically a valid configuration for the internal engine, but without the "provider" to tell it how to close HTML tags, it does nothing.

Revert the changes

Get rid of all the code we added above. It is good to go through the process of adding code and seeing that it doesn't work as expected. It is also good to know that there is a way to enable auto-closing tags in Monaco, even if it's not as simple as adding a property to the editor configuration.

The Simplified Solution

So what can we do? We will add a keydown event listener to the HTML editor that will automatically close the tag when the user types a closing angle bracket.

app.js 
editors.html.onKeyDown((event) => {
  if (event.browserEvent.key === '>') {
    const model = editors.html.getModel();
    const position = editors.html.getPosition();
    const lineContent = model.getLineContent(position.lineNumber).substring(0, position.column - 1);
    const match = lineContent.match(/<(\w+)$/);
    if (match) {
      const tagName = match[1];
      const closingTag = `</${tagName}>`;
      setTimeout(() => {
        const newPos = editors.html.getPosition();
        editors.html.executeEdits("auto-close", [{
          range: new monaco.Range(newPos.lineNumber, newPos.column, newPos.lineNumber, newPos.column),
          text: closingTag,
          forceMoveMarkers: false
        }]);
        editors.html.setPosition(newPos);
      }, 0);
    }
  }
});

What This JavaScript Does:

I think this one warrants a line by line explanation:

  • editors.html.onKeyDown((event) => { ... }); : This attaches an event listener to the HTML editor that triggers every time a key is pressed.
  • if (event.browserEvent.key === '>') { ... } : It checks if the key pressed was the greater-than symbol (>), which is used to close HTML tags.
  • const model = editors.html.getModel(); : Gets the current document model (the text content) of the editor.
  • const position = editors.html.getPosition(); : Gets the current cursor position (line and column number).
  • const lineContent = ... : Extracts the text on the current line up to the cursor position.
  • const match = lineContent.match(/<(\w+)$/); : This is the core logic. It uses a regular expression to look for an opening HTML tag at the end of the line.
    - < : Matches the opening bracket.
    - (\w+) : Matches and captures one or more "word" characters (letters, numbers, underscore). This is the tag name (e.g., "div", "h1", "p").
    - $ : Anchors the match to the end of the line.
  • if (match) { ... } : If the regular expression finds a match (meaning the user just typed an opening tag), the code inside the if-block executes.
  • const tagName = match[1]; : Extracts the captured tag name from the match result.
  • const closingTag = `</${tagName}>`; : Creates the corresponding closing tag (e.g., if tagName is "div", closingTag becomes "</div>").
  • setTimeout(() => { ... }, 0); : This is a clever trick. It schedules the code to run after the current event loop finishes. This ensures that the user's typed '>' character is already in the editor before we try to insert the closing tag.
  • const new = editors.html.getPosition(); : Gets the current cursor position (line and column number) after the '>' has been added to the editor.
  • editors.html.executeEdits("auto-close", [{ ... }]); : This is the command that actually modifies the editor content.
    - "auto-close" : An identifier for this edit operation.
    - range: new monaco.Range(...) : Defines the exact position where the text should be inserted. It's set to the position immediately after the typed '>'.
    - text: closingTag : The text to insert (the closing tag).
    - forceMoveMarkers: true : Ensures that any selection or cursor position is correctly adjusted after the edit.
  • editors.html.setPosition(newPos); : After inserting the closing tag, this moves the cursor to the new position (right after the '>'), so the user can continue typing immediately without having to manually move the cursor.
