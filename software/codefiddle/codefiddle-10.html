📋 Part 10

In this part of the tutorial, we will fix a bug where the console and timestamp settings are not being saved correctly. When you open and close the console, we are showing incorrectly duplicated logs.

The Problem

There are two connected issues causing the duplicate logs and making it look like the settings were broken:

  1. The Array isn't Resetting: When clearConsole() is called, it correctly clears the user interface (innerHtml = ''), but it doesn't clear the consoleMessages array stored in memory.
  2. Re-running Code Unnecessarily: When you toggle the "Show/Hide Console" button, it unnecessarily calls updatePreview(). This function forces the iframe to re-run the code inside the editor. Because the consoleMessages array isn't being cleared, it appends an entirely new identical set of logs to the history. Thus, every time you open or close the console it adds more and more duplicate logs.

The Fix

The fix is to ensure that the consoleMessages array is cleared whenever the console is cleared. Additionally, we need to remove the unnecessary call to updatePreview() when toggling the console visibility. This ensures that the console is only re-rendered when the code is actually run.

function clearConsole() {
  consoleMessages.length = 0; // Fix: Clear the stored messages array
  consolePanel.innerHTML = '';
//...

function toggleConsole() {
//...
  toggleConsoleBtn.textContent = consoleVisible ? 'Hide Console' : 'Show Console';
  // Fix: Removed updatePreview();
}
