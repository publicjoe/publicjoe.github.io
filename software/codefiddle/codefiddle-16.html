Build Your Own Web-Based Code Editor with Live Preview

📋 Part 16

In this part of the tutorial, we will add some more configuration to the four pages. We will also make some tweaks to the tagging solution as some tags that do not need a closing tag are being added.

What configuration

We will add the following configuration to the four pages:

  • autoClosingBrackets: 'always' : Automatically closes brackets when the user types an opening bracket.
  • matchBrackets: 'always' : Automatically matches brackets when the user types an opening bracket.
  • tabSize: 2 : Sets the tab size to 2 spaces.
  • insertSpaces: true : Inserts spaces instead of tabs.
  • detectIndentation: false : Disables automatic indentation detection.
  • formatOnType: true : Automatically formats the code when the user types.
  • formatOnPaste: true : Automatically formats the code when the user pastes code.

Now we could add the same code to each editor, but that would mean if we wanted to change the code then we would need to update it in four places every time. This could lead to errors and inconsistencies.

Instead, we can separate out the code to an object, then build a helper function that will be used to create each editor.

So go ahead and delete the four editors and replace with the following code:

app.js 
const commonOptions = {
  theme: 'vs-dark',
  automaticLayout: true,
  autoClosingBrackets: 'always',
  matchBrackets: 'always',
  tabSize: 2,
  insertSpaces: true,
  detectIndentation: false,
  formatOnType: true,
  formatOnPaste: true
};

function createEditor(id, language, value) {
  return monaco.editor.create(document.getElementById(id), {
    ...commonOptions,
    language: language,
    value: value
  });
}

editors.html = createEditor('html-editor', 'html', "<h1>Welcome to CodeFiddle!</h1>\n<p>Start editing to see live changes.</p>");
editors.css  = createEditor('css-editor', 'css', "h1 { color: dodgerblue; }");
editors.js   = createEditor('js-editor', 'javascript', "console.log('Hello from JavaScript');");
editors.ts   = createEditor('ts-editor', 'typescript', "let message: string = 'Hello from TypeScript';\nconsole.log(message);");

What This JavaScript Does:

commonOptions

This is an object that contains all the common options that will be used for each editor. This is a good way to keep your code DRY (Don't Repeat Yourself).

createEditor

This is a helper function that will create an editor with the given options. This is a good way to keep your code DRY (Don't Repeat Yourself).

editors

This is the rebuild of the editors using the helper function. Notice how we are using the spread syntax (...) to pass the common options to the createEditor function.

Improving the Tagging

The current tagging solution is not perfect and it is not very efficient. So let's improve it. So what we will do is to add a list of self-closing tags to our app.js file. We will add this to the editors.html.onKeyDown function.

app.js 
// Existing code
const tagName = match[1];

// Add this list of self closing tags
const selfClosingTags = [
  'area', 'base', 'br', 'col', 'embed', 'hr', 'img', 'input',
  'link', 'meta', 'param', 'source', 'track', 'wbr'
];

if (selfClosingTags.includes(tagName.toLowerCase())) {
  return;
}

What This JavaScript Does:

This list is used to check if the tag is a self-closing tag. If it is, then we don't add a closing tag.
