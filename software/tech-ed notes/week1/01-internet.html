Workshop

Definitions

The Internet is a global network of computers, connecting like a spider’s web. To communicate over this network you have to follow certain rules, or “protocols”. You must send the correct information at the correct times or you won’t get a reply.

There are three primary protocols that work together to allow us to visit and download a website:

  • Internet Protocol (IP), provides every connected computer with an “IP Address”, like a postcode: 0.0.0.0, or 1.2.3.4, up to 255.255.255.255.
  • Domain Name System (DNS), maps a website address or “domain name” to an IP address: google.com -> 42.250.200.46, like your phone’s contact list.
  • Hyper Text Transfer Protocol (HTTP), handles transferring website code to a visitor’s computer for display

Loading a web page

When you type in google.com:

  • web browser (the “client”) asks the DNS for the IP address the domain owner has specified.
  • DNS returns: 42.250.200.46
  • web browser (“client”) makes an “HTTP Request” to that address.
  • the computer at the address (the “web server”) returns an “HTTP Response”, containing the website code
  • web browser (“client”) displays the website

Together these protocols enable what we consider the modern web. Mobile apps use exactly the same stack, except the data is shown in custom software instead of a web browser.

Understanding URL(s)

Understanding URL(s) URL stands for Uniform Resource Locator, and you can think of it as the address of something on the internet, just like the address of your house. The structure of an URL can be broken down into a few key parts - first, there’s a ‘protocol’, which lets the browser know a bunch of rules about how to get the resources at that location - you’ve been using ones without knowing it; HTTP (Hypertext-Transfer-Protocol) and HTTPS (secure)

After the protocol is the ‘domain name’ which is the name of the website - like google.com or youtube.com.

Next, you may see a “path,” which specifies the exact resource on the website, like a specific page or image. For instance, in ‘www.example.com/about’, ‘/about’ is the path that takes you directly to the About page of the site.

Finally, you may also see ‘query’ strings at the end of some URLS. This part of a URL is optional and starts after a question mark (?). Each parameter in a query string is made up of a key and a value, connected by an equals sign (=). Multiple parameters can be included in a single URL and are separated by an ampersand (&). For example, in the URL www.example.com/products?category=books&sort=ascending, category=books and sort=ascending are two parameters that tell the server to show products categorised under books and to sort them in ascending order.

My answers localhost is used in the URL to refer to the current computer, using the IP address 127.0.0.1. So http://localhost:5500/dev/workshops/01-html-tags/index.html works the same as writing http://127.0.0.1:5500/dev/workshops/01-html-tags/index.html. A nameserver translates domain names into IP addresses. Simply put, they translate human-readable names and computer-readable IP addresses. They are needed because humans cannot remember IP addresses easily, but we can remember names easily. A client is typically a browser that requests a html page from a server. The server holds the web pages. Communication between clients and servers is done by requests and responses: A client sends a HTTP(S) request to the server The server receives the request The server then processes the request to find the page, if it can The server returns a HTTP response to the browser (the page or an error) The client receives the response and displays the page or error to the user A port is a virtual endpoint for communicating with a computer. It is identified by a 16-bit number (0 to 65535). This allows applications and services to exchange data. The default port for the HTTP protocol is 80.
