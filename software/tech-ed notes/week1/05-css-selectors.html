Essential Selectors

You‚Äôll use these all the time, for applying layout and design to your pages.

  • element - Selects all elements of a given type, eg div or p
  • .class - Selects all elements with a given class, eg .container or .box
  • #id - Selects the element with a given id, eg #header or #footer

You can also target multiple elements, classes or ids with the same rules if you need to (to avoid repeating yourself)

  • element1, element2 - Selects all elements of type element1 and all elements of type element2, eg div, p or div, .box

Additional Selectors

You can be more specific with your selectors by selecting elements that are inside other elements, or elements of a particular type that also have a particular class or id.

  • element.class - Selects all elements of a given type with a given class, eg div.container or p.box
  • element#id - Selects the element of a given type with a given id, eg div#header or p#footer
  • element1 element2 - Selects all elements of type element2 that are inside an element of type element1, eg div p or div .box
  • element1 > element2 - Selects all elements of type element2 that are direct children of an element of type element1, eg main.content > p or #mylist > .box
  • element1 + element2 - Selects all elements of type element2 that are immediately after an element of type element1, eg article + p or div + .box
  • element1 ~ element2 - Selects all elements of type element2 that are after an element of type element1, eg div ~ p or .sidebar ~ .box

Pseudo Selectors

Pseudo selectors allow you to target elements in a particular state. A use of these you‚Äôll recognise is using :hover to allow you to apply a different colour or style to an element only when the user‚Äôs mouse is hovering over it.

  • element:hover - Selects all elements of type element that are currently being hovered over, eg div:hover or p:hover
  • element:active - Selects all elements of type element that are currently being clicked on, eg div:active or p:active
  • element:disabled - Selects all elements of type element that are currently disabled, eg button:disabled.

Structural Selectors

Structural selectors allow you to target elements based on their position in the document.

  • element:first-child - Selects all elements of type element that are the first child of their parent, eg div:first-child or p:first-child
  • element:last-child - Selects all elements of type element that are the last child of their parent, eg div:last-child or p:last-child
  • element:nth-child(n) - Selects all elements of type element that are the nth child of their parent, eg div:nth-child(2) or p:nth-child(2)
  • element:nth-last-child(n) - Selects all elements of type element that are the nth child of their parent, counting from the last child, eg div:nth-last-child(2) or p:nth-last-child(2)
  • element:nth-of-type(n) - Selects all elements of type element that are the nth element of their type, eg div:nth-of-type(2) or p:nth-of-type(2)

Other Selectors

There are a few other selectors that are useful in certain situations.

  • element::before - Selects the content before an element, eg div::before or p::before

Eg - if we have the h2 below, and do h2::before it will ‚Äòselect‚Äô the space between >s. (the closing tag and the H - )

<h2>sunshine</h2>
  • element::after - Selects the content after an element, eg div::after or p::after
  • element::first-letter - Selects the first letter of the content of an element, eg div::first-letter or p::first-letter
  • element::first-line - Selects the first line of the content of an element, eg div::first-line or p::first-line
  • element::selection - Selects the content of an element that is currently selected, eg div::selection or p::selection
