In this part of the tutorial, we enhance our mini code editor by adding a "Clear Console" button. This button allows users to clear the console panel, making it easier to manage log messages and errors.

Preview Toolbar with Clear Button (index.html)

We will add a button to a preview toolbar that, when clicked, clears all messages from the console panel.

Add the following code after the line <div id="right-panel"></div>

index.html 
<div id="preview-toolbar">
  <button id="clear-console-btn">Clear Console</button>
</div>

What This HTML Does:

  • Preview Toolbar: A new <div> with the ID preview-toolbar is created to hold the toolbar elements.
  • Clear Console Button: A button with the ID clear-console-btn is added. This button will be used to clear the console panel when clicked.

CSS Styling Changes (styles.css)

We will add some CSS to style the preview toolbar and the clear console button.

Add the following CSS to your styles.css file after the #right-panel section

styles.css 
/* Toolbar above preview */
#preview-toolbar {
  position: sticky;
  top: 0;
  z-index: 10;
  display: flex;
  justify-content: flex-end;
  align-items: center;
  background: #1e1e1e;
  padding: 4px 8px;
  border-bottom: 1px solid #333;
}

#preview-toolbar button {
  background: #333;
  color: #eee;
  border: none;
  padding: 4px 8px;
  border-radius: 4px;
  cursor: pointer;
  font-size: 0.85rem;
  transition: background 0.2s;
}

#preview-toolbar button:hover {
  background: #444;
}

This will place the ‚ÄúClear‚Äù button in a slim toolbar above the preview pane, matching the dark theme of the rest of the layout.

Let's explain what this is doing:

#preview-toolbar: This styles the toolbar to be sticky at the top of the preview panel, with a dark background and padding for better appearance.

position: sticky; keeps the toolbar visible when scrolling within the preview panel.

top: 0; ensures it sticks to the top.

z-index: 10; ensures it stays above other elements.

display: flex; and related properties align the button to the right side of the toolbar.

 

#preview-toolbar button: This styles the button to match the overall dark theme, with padding, border-radius, and hover effects for better user experience.

background: #333; gives the button a dark background.

color: #eee; makes the text light for contrast.

border: none; removes the default button border.

padding: 4px 8px; adds space inside the button.

border-radius: 4px; rounds the button corners.

cursor: pointer; changes the cursor on hover to indicate it's clickable.

transition: background 0.2s; adds a smooth transition effect when hovering.

 

#preview-toolbar button:hover: This changes the button background color when hovered over, providing visual feedback to the user.

background: #444; makes the button slightly lighter on hover.

What This CSS Does:

  • Sticky Toolbar: The toolbar is styled to be sticky at the top of the preview panel, ensuring it remains visible when scrolling.
  • Button Styling: The clear console button is styled to match the dark theme of the editor, with hover effects for better user experience.

JavaScript Functionality (script.js)

We will add JavaScript to handle the click event on the "Clear Console" button, which will clear all messages from the console panel.

Add the following code to your script.js file, preferably after clearConsole function:

script.js 
// Attach Clear Console button
document.getElementById('clear-console-btn').addEventListener('click', clearConsole);

What This JavaScript Does:

  • Event Listener: An event listener is attached to the button with the ID clear-console-btn. When the button is clicked, it triggers the clearConsole function.
