Hoisting

Variable Hoisting

Hoisting is JavaScript's default behavior of moving declarations to the top of the functional scope within the javascript interpreter. What this really means is that all function and variable declarations are added to the memory inside a JavaScript data structure called Lexical Environment before the code is executed.

As a programmer, this means that a variable can be declared after it has been used.

Example

function getShape(condition) {
    // shape exists here with a value of undefined
    console.log(shape); // OUTPUT : undefined

    if (condition) {
        var shape = "square";
        // some other code
        return shape;
    } else {
        // shape exists here with a value of undefined
        return false;
    }
}

In ES6, variables defined with let are NOT hoisted to the top.

Using a let variable before it is declared will result in a ReferenceError.

Example

console.log(hoist); // Output: ReferenceError: hoist is not defined ...
let hoist = 'The variable has been hoisted.';

We should declare, then assign our variables to a value before using them.

Example

let hoist = 'The variable has been hoisted.';
console.log(hoist); // Output: The variable has been hoisted.

In a similar manner to let, a const variable cannot be used before it has been declared.

Variables declared using const are treated as constants, and therefore they cannot be re-assigned values once defined. Due to this, every const declaration must be initialized at the time of declaration.

So if we look at the first example, and use let instead of var, then shape only exists within the if block.

function getShape(condition) {
    // shape doesn't exist here
    console.log(shape); // ReferenceError: shape is not defined

    if (condition) {
        let shape = "square";
        // some other code
        return shape;
    } else {
        // shape doesn't exist here either
        return false;
    }
}

Function Hoisting

Functions in javascript are created through either Function Declaration or Function Expression.

A function declaration defines a function with the specified parameters. In JavaScript, function declarations hoist the function definitions, and therefore, these functions can be used before they are declared.

hoisted() // output: "Hoisted"

function hoisted() {
    console.log('Hoisted')
}

The interpreter hoists the function declaration to the top of the functional scope.

A function that is created using a function expression is NOT hoisted. Therefore, you cannot use function expressions before defining them.

Example

const myFunction = function [name](param1, param2, ...) { [statements] }

The [name] is optional, therefore these can be anonymous functions. We can alsso use arrow functions:

const myFunction = (param1, param2, ...) => { [statements] }
